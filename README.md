# Eldergenix

**Founding AI product engineer and technical AI product manager building autonomous agent systems, MCP tooling, bio-AI infrastructure, and production-grade AI workflows.**

I build AI products where agents do useful work inside real engineering, research, and business loops: retrieval, tool use, workflow orchestration, evaluation, observability, runbooks, developer automation, genomics, scientific discovery, and full-stack product delivery.

My strongest work sits at the intersection of **0-to-1 AI product engineering**, **agentic systems**, **high-trust biomedical workflows**, and **technical product ownership**. I scope POCs and MCPs, define product requirements, map user workflows, own roadmaps, build full-stack systems, integrate external tools/APIs, coordinate engineering teams, and ship AI workflows that can be tested, reviewed, launched, and improved.

## How I Work

| Principle | Operating behavior |
| --- | --- |
| Own the ambiguous middle | I am strongest where product intent, user needs, technical architecture, and delivery risk are not yet cleanly separated. |
| Build from workflow reality | I start with customer discovery, user workflow mapping, product requirements, and the concrete job the AI system must perform. |
| Ship useful systems, not demos | I prioritize production paths, release readiness, feedback loops, observability, documentation, and technical debt tradeoffs. |
| Treat agents as systems | I design around tool boundaries, retrieval quality, structured outputs, approvals, dry runs, postcondition checks, and evaluation loops. |
| Keep humans in the loop where it matters | I build for reviewability, traceability, uncertainty handling, evidence grounding, and safe handoffs between model output and system action. |
| Communicate across functions | I work across founders, customers, engineers, users, marketing, leadership, investors, scientists, and clinicians to keep scope, risks, and outcomes visible. |
