## Ongoing Workflow Rules

Use this workflow for flagship repos going forward:

1. Create a GitHub issue before a meaningful feature or cleanup.
2. Work on a branch.
3. Open a pull request, even for solo work.
4. Include screenshots, logs, or test output in the PR body.
5. Merge into the default branch so commits count on the contribution graph.
6. Create releases for stable demos or recruiter-ready milestones.

## Contribution Graph Checks

If real work is missing from the contribution graph, check these first:

- The commit author email must be connected to the GitHub account.
- Commits count when they land on the default branch or `gh-pages`.
- Private contributions need to be enabled in GitHub profile contribution settings.
- Fork commits usually need to be merged upstream before they count as profile contributions.
